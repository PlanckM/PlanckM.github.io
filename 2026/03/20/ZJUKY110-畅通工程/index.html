二、代码核心实现思想

1. 核心算法：并查集（Disjoint Set Union，DSU）

这段代码基于并查集算法实现，核心解决的是无向图连通分量统计问题，最终求解：给定n个节点和m条已有的边，至少还需要添加多少条边，才能让所有节点变成一个连通整体。这是并查集最经典的应用场景之一，常用于连通性判断、集合合并、最小修路/搭桥问题。

2. 核心变量作用

  • parent数组：采用向量动态存储，下标对应节点编号，存储每个节点的父节点；初始值为-1，代表该节点是所在集合的根节点，没有父节点。
  • setCount变量：实时统计当前图中的连通分量总数，初始时每个节点独立，总数为n，每成功合并两个不同集合，总数减1。

3. 核心函数逻辑

（1）Find查找函数

作用是找到某个节点所在集合的根节点，根节点是判断两个节点是否属于同一集合的唯一依据。 实现逻辑：从目标节点开始，循环向上追溯父节点，直到找到父节点为-1的节点，即为根节点；当前代码是基础版查找，未做路径压缩，逻辑简单易懂，适合入门理解。

（2）Union合并函数

作用是合并两个不同的集合，实现连通分量的合并。 实现逻辑：先分别查找两个节点的根节点，若根节点不同，说明分属不同连通分量，将其中一个根节点的父节点指向另一个根节点，完成合并，同时连通分量总数减1；若根节点相同，说明已连通，不做任何操作。

4. 主函数整体执行流程

  1. 多组用例处理：通过while循环支持多组测试数据输入，输入n=0时终止，适配算法题常见的多组输入要求。
  2. 初始化操作：每次新用例都重置parent数组，全部赋值为-1，同时初始化连通分量数为n（每个节点单独成组）。
  3. 边的合并：遍历m条已有的边，每读一条边就调用合并函数，将两个节点所在集合连通，同步更新连通分量总数。
  4. 结果计算：数学结论：把k个独立连通分量连成一个整体，最少需要k-1条边，直接输出setCount-1即可；额外判断setCount=0的边界情况，避免输出异常。
