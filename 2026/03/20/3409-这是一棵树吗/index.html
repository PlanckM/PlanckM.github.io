3409. 这是一棵树吗

一、带逐行详细注释的C++代码

二、代码实现核心思想

1. 题目核心需求

这段代码用于判定给定的有向图是否为有向树，支持多组测试用例，输入以0 0结束单组用例，以-1 -1结束整个程序，最终输出每组用例是否为合法有向树。有向树区别于普通无向树，有严格的有向图规则，需同时满足多项核心条件，代码通过分步校验+并查集判环实现精准判定。

2. 核心变量作用

  • vector v：标记顶点是否出现，避免统计未使用的顶点，精准计算当前用例的有效顶点数，防止无效遍历干扰判断。
  • vector parent：并查集父节点数组，搭配路径压缩优化，核心用于判断连接边后是否形成无向环，有向树不允许存在任何环路。
  • vector indegree：入度数组，记录每个顶点的入度，是有向树判定的核心指标，限制每个顶点入度不超过1，且仅有一个根节点入度为0。
  • V_num、E_num：分别统计有效顶点数和有效边数，用于验证树的核心数学性质。
  • is_tree：布尔标记位，全程跟踪判定结果，任意一步不满足条件则立即置为false，提前剪枝减少无效计算。

3. 核心算法与逻辑流程

（1）并查集基础功能

采用带路径压缩的并查集，相比普通并查集查找效率更高，递归查找根节点的同时，将路径上所有节点直接指向根节点，缩短后续查找路径；初始化时每个节点父节点为自身，独立成集合，合并时将子节点根指向父节点根，实现集合合并。

（2）分步判定逻辑（全程剪枝，提前判错）

代码分为边读取实时判定和最终全局判定两步，实时判定提前排除明显不合法的情况，减少后续计算量：

  1. 实时入度校验：每条有向边a→b，b的入度自增，若任意顶点入度超过1，直接判定非树（有向树每个节点最多有一个父节点）。
  2. 实时环判定：通过并查集查找两个端点根节点，根节点相同说明连接后会形成环路，直接判定非树，无环则合并集合。
  3. 空树特殊处理：无任何顶点和边的空图，属于合法的树。
  4. 树的数学性质校验：有效顶点数必须等于有效边数+1，这是树的核心性质（连通且无环的图必满足），不满足则直接排除。
  5. 根节点唯一性校验：遍历所有出现过的顶点，统计入度为0的节点数，有向树必须恰好有1个根节点（0个则存在环，多个则森林非树）。

4. 整体执行流程

  1. 多组用例循环：无限循环处理测试用例，输入-1 -1时退出程序，每组用例开始前重置所有数组和变量，避免用例间数据干扰。
  2. 边读取处理：循环读取每条有向边，统计顶点数和边数，同步做入度和环的实时判定。
  3. 最终条件校验：通过实时判定后，再校验空树、顶点边数关系、根节点数量三大核心条件。
  4. 结果输出：按用例编号输出是否为合法有向树的结论。

三、关键注意事项

\1. 节点范围：代码预设节点范围1~10000，适配常规算法题的节点大小限制，无需动态扩容。

\2. 空树判定：空图（无顶点无边）属于合法树，是特殊边界情况，代码单独做了处理。

\3. 提前剪枝：任意一步不满足条件就将is_tree置为false，后续跳过无效判断，提升运行效率。

\4. 有向树vs无向树：有向树额外要求入度限制和单一根节点，不能只用无向树的判定规则，这是代码核心区分点。
