3405. W的密码

代码实现思想 #### 核心目标 对输入字符串按字符范围拆分为3个子串，分别执行指定次数的循环右移，再将移位后的子串字符放回原始位置，生成最终字符串。 #### 实现步骤（分4步） 1. 输入处理与初始化 - 循环读取移位参数 k1/k2/k3 和原始字符串 in，直到输入三个0终止。 - 每次处理新输入前清空全局变量，避免上一次数据干扰。 2. 字符串拆分（核心预处理） - 遍历原始字符串的每个字符，按范围分类： - a-i → 存入 s1，并记录字符在原始字符串中的下标 → s1_pos； - j-r → 存入 s2，下标记录 → s2_pos； - s-z → 存入 s3，下标记录 → s3_pos。 - 作用：将不同范围的字符分离，方便后续独立移位。 3. 循环右移处理 - 对 s1/s2/s3 分别执行循环右移： - 先对移位次数取模（k % 长度），避免移位次数超过字符串长度（如长度为3，移4位等价于移1位）； - 利用字符串切片 substr 实现循环右移：末尾k个字符 + 开头(长度-k)个字符。 - 例：s1 = "abc"，k1=2 → 右移后为 "cab"。 4. 字符还原位置 - 遍历原始字符串的每个下标位置，按 s1_pos/s2_pos/s3_pos 记录的原始下标，将移位后的 s1/s2/s3 字符依次放回对应位置，最终拼接成结果字符串。 ### 总结 1. 核心逻辑：拆分（按字符范围）→ 移位（循环右移）→ 还原（按原始下标）。 2. 关键技巧：用 substr 实现循环右移，用下标数组记录字符原始位置，保证移位后字符能精准归位。 3. 优化点：移位次数取模避免无效移位，每次输入前清空全局变量防止数据残留。
